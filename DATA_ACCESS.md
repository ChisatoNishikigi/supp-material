
# Data Access

This repository is intended as an anonymous supplementary resource for the submission.

It provides:

- dataset metadata,
- source composition summaries,
- source-level references,
- access notes, and
- supplementary documentation for the dataset setting used in the paper.

## What This Repository Provides

This repository may include:

- metadata tables,
- source-level links to official pages, project pages, dataset pages, or community entry points,
- split usage notes,
- source composition summaries, and
- example file structures or manifests.

## What This Repository Does Not Provide

This repository does **not** aim to redistribute all video files used in the experiments.

In particular, this repository does not generally provide:

- full copies of third-party datasets,
- direct redistribution of all collected videos,
- signed or temporary media URLs,
- platform-restricted attachment links, or
- materials whose redistribution may be restricted by the original source, hosting platform, or dataset terms.

## Access Principles

The dataset setting in the submission includes materials from multiple source types. Different source types are handled differently in this repository.

### 1. Public Dataset Sources

For publicly released datasets, this repository provides source-level references and metadata notes.

Users should obtain the original data from the corresponding official dataset pages and follow the original dataset terms, licenses, and access conditions.

### 2. Official Public Pages and Project Pages

For videos associated with official product pages, project pages, research pages, or public showcases, this repository provides source-level references rather than direct redistribution of media files.

These references are intended to document provenance at the source level.

### 3. Community-Hosted Sources

For sources hosted through community platforms, this repository provides source-level entry points only.

Access to such materials may depend on the hosting platform, server visibility, channel permissions, or other platform-specific conditions. This repository does not redistribute platform-hosted video attachments.

## Rationale

The repository is designed to support transparency of dataset construction while respecting source boundaries.

For this reason, the repository emphasizes:

- provenance documentation,
- source-level traceability,
- metadata release, and
- access notes

rather than full redistribution of heterogeneous video materials collected from different origins.

## Notes on Reuse

Users of the referenced sources should consult the original source pages, dataset pages, or hosting platforms for the applicable terms of use, licensing conditions, and access policies.

The presence of a source-level reference in this repository should not be interpreted as a statement that all associated media files may be freely redistributed.

## Summary

In short, this repository is intended to document **where the data come from** and **how the dataset setting is organized**, rather than to serve as a full mirror of all video materials used in the experiments.
